RAG Best Practices
click a card for the full rationale
🛡️
Five-Layer Guardrails

Defense in depth: input gate → hardened prompt → RAG-only grounding → output validator → ops limits.

🎯
RAG-Only Grounding

Give the model the smallest right context — not the most. Nothing off-topic to leak; a jailbreak returns "I don't have that."

🚧
Prompt-Injection Defense

Fence retrieved content in XML tags marked untrusted="true"; sanitize corpus at index time.

✅
Faithfulness / Anti-Hallucination

Every number in the answer must appear in LIVE_DATA, or the response is blocked. Cite resort + asOf timestamp.

📏
Token Budget & Cost Control

Hard-cap context ≤ 2,500 tok, output ≤ 350. ~$0.003 / request. Cheap 8B for classify, 70B for the answer.

📊
Observability & Ops

Log every request (tokens, cost, latency, retrieval scores, blockReason). Rate-limit, alarm, kill-switch.
